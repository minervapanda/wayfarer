# Worklog — Builder 2 (Capture pipeline: exif.js, ingest.js, geocode.js)

- **2026-07-11** — Replaced all three capture-pipeline stubs; contracts per ARCHITECTURE.md §3 unchanged.
  - **js/exif.js**: ported `parseExifBuffer` + `readExif` from legacy v1. Dependency-free JPEG APP1/TIFF parser, both endiannesses ('II'/'MM'); GPS DMS rationals + N/S/E/W refs → signed decimals (with range/(0,0) sanity checks); DateTimeOriginal 0x9003 (Exif sub-IFD) with 0x0132 fallback → ISO; also surfaces Orientation 0x0112 (documented extra field, default 1) for ingest. Hardened vs legacy: keeps scanning past non-Exif APP1 segments (XMP), bounds-checked IFD reads, only reads first 512 KB of the file. Never throws — nulls for PNG/HEIC/malformed. Verified with synthetic JPEG fixtures in Node: LE + BE, S/W refs, XMP-before-EXIF, truncated/empty/garbage inputs — all green.
  - **js/ingest.js**: `ingestFiles` filters to `image/*`, caps at 24/call with a `'toast'` bus warning, reads EXIF off the ORIGINAL bytes first (re-encode strips it), then canvas-downscales to max 1600 px long edge, JPEG q0.8, white-flattened (transparent PNGs). Orientation preserved: browsers with `image-orientation: from-image` (feature-detected via `CSS.supports`) bake it in at decode; older engines get the manual 8-case transform. Undecodable files (HEIC on non-Safari, corrupt) are skipped silently; returns `[]` when nothing usable; blobRecs NOT persisted (compose controller saves).
  - **js/geocode.js**: ported legacy Nominatim reverse/forward and upgraded: serialized queue ≥1100 ms between request starts; in-memory + `localStorage['wayfarer-geocache-v1']` cache (reverse keyed by 4-dp rounded coords, forward by lowercased query, 300-entry LRU-ish eviction, private-mode safe); in-flight dedupe; English via `accept-language=en` query param (kept custom headers to `Accept` only per ground rules); 10 s abort timeout; `navigator.onLine === false` short-circuits; `#osm-attribution` gets `ATTRIBUTION` (textContent) after first success. Never throws — null on any failure, app stays fully offline-capable. Verified with mocked-fetch Node tests: rate-limit gap, dedupe, cache hits, error/offline paths — all green.
  - `node --check` clean on all three files. No requests needed outside my files.
