## 2026-07-12 — Story-card renderer (js/storycard.js, new)

- Created `js/storycard.js`: pure canvas-2D renderer, no DOM UI, no external libs. Exports the frozen contract exactly: `FORMATS` (story 1080×1920 / square 1080×1080 with labels) and `renderStoryCard(entry, format='story') -> { blob (image/png), width, height, filename }`.
- Filename: `wayfarer-<slug>-<dateISO>.png` — slug from location name, else title, else `entry`; NFKD-normalized, diacritics stripped, `undated` fallback when no valid date.
- Paper ground samples live CSS tokens (`--paper`, `--paper-edge`, `--ink`, `--ink-soft`, `--accent`, `--tape`, `--line`, `--stamp`) via getComputedStyle each render (theme switches respected), with exact passport-theme hex fallbacks. Grain = ~3.4k 1px speckles from a mulberry32 PRNG seeded by FNV-1a(entry.id) — re-renders are pixel-identical. Plus warm wash, deckled-edge gradient bands + jittered hairline, gentle vignette, and a whisper of grain over the finished composition.
- Photos: up to 4 via `store.getBlob`; `createImageBitmap` with `<img>`+`util.blobUrl` fallback (cached URL, never revoked here); undecodable blobs (HEIC etc.) skipped and the layout adapts to the actual count. Curated layouts: 1 hero polaroid, 2 overlapping diagonal, 3 fanned stack + offset, 4 loose 2×2 — deterministic ±2–6° rotations and jitter, white polaroid frames with chin, soft ctx shadows, cover-crop, striped washi-tape strip (offscreen pattern canvas) on the topmost photo. Wide zones spread placements horizontally.
- Zero photos: postcard motif — lighter rotated card with center divider, four ruled address lines, perforated postage stamp bearing the location initial in `--stamp`, circular postmark with wavy cancellation lines, script greeting (location or "wish you were here…").
- Text: bold serif title (Georgia/Iowan stack, 64px story, +15% when no photos), 2-line max with ellipsis; small-caps letter-tracked dateline `APR 3, 2026 · KYOTO, JAPAN` with a drawn map-pin glyph; story excerpt trimmed to ≤260 chars at a word boundary, manual measureText word-wrap, ≤7 lines with dynamic reduction so photos keep a minimum zone; excerpt block skipped entirely when story is empty. Footer "— from my Wayfarer diary" in Snell Roundhand only if `document.fonts.check` confirms it, else italic serif.
- Instagram safe area honored: story content confined to y 220–1700; square uses balanced 88px margins.
- Robustness: `renderStoryCard` never throws — outer catch falls back to a minimal typographic card; `toBlob` null falls back to dataURL→bytes; missing entry fields all degrade. No HTML produced anywhere (fillText only).
- Verified: `node --check js/storycard.js` passes; pure helpers (PRNG determinism, filename slugging, excerpt trim) smoke-tested in node.

## 2026-07-12 — Orientation-aware collage frames (js/storycard.js)

- Frames now follow each photo's own shape. Every decoded bitmap is classified from its pixel aspect (w/h): `portrait` < 0.85, `landscape` > 1.18, else `square` — the shared thresholds for this round. Per class the polaroid's PHOTO AREA is exactly 3:4 / 4:3 / 1:1 (`ORIENT_ASPECT` + `printDims()`); polaroid chrome is untouched (5% pad, 18% chin of frame width, same shadow/tape/hairline), and `coverCrop` stays for the residual sliver-trim — photos are never stretched or letterboxed.
- New `layoutCollage(photos, zone, rng)` replaces `drawCollage` + `SIZE_COEF`: it anchors prints from the curated fraction specs (jitter + rotation drawn from the seeded rng in the same order as before, so re-renders stay pixel-identical per entry), sizes all prints by one shared photo-area unit (equal visual weight across mixed orientations), caps it per count (`UNIT_CAP`), then shrinks it so every ROTATED frame's extents stay inside the photo zone — prints can no longer poke into the title/excerpt/footer bands or out of the story 220–1700 / square margin safe areas.
- Count/mix layouts: 1 photo = hero shaped to its class (landscape spans ~90% of the zone width, portrait runs tall, square in between); 2 photos pick by pair (`LAYOUTS_2`): two portraits side-by-side, two landscapes stacked with a sideways offset, mixed/square staggered diagonal overlap; 3 keeps the fanned stack and 4 the loose 2×2, both with per-print class shapes.
- Dynamic vertical banding: the excerpt-line-reduction loop now biases the reserved photo band by mix (all-landscape ×0.85 — frees text lines; all-portrait ×1.15 — buys height); after layout, the collage's TRUE rotated bbox is measured and the collage + excerpt group is re-centered in the band, with the excerpt hugging the real collage bottom (`exFirstBaseline`), so text and prints never collide in any mix while the footer stays anchored.
- Untouched: `FORMATS`/`renderStoryCard` contract, minimal-fallback card, zero-photo postcard branch, ImageBitmap close-in-finally cleanup, excerpt/dateline ellipsis rules, filename slugging, token sampling.
- Verified: `node --check` passes; VM harness (scratchpad `layout-test.mjs`) exercised 15 orientation mixes × 3 zones asserting zone containment of rotated prints, exact class aspect of each photo window, classification thresholds (0.84→portrait, 0.85/1.18→square, 1.19→landscape), determinism of repeated layouts, and the three 2-photo arrangements — all passed.
